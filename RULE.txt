AddObject 규칙
- 게임 오브젝트의 객체는 해당 클래스의 Create 함수에서만 생성할 수 있다.
- 포함되어있는 Scene이 있어야만 한다.
- Create 함수에서 AddGameObject를 해야한다.
- Scene에 다른 컨테이너가 있을 경우 해당 씬에서 관리한다.


Init 규칙
- 객체 준비
- 다른 객체의 정보를 가져오는 등의 내용은 Init에서 작성
- Reset 호출

Reset 규칙
- 객체 초기화
- 상속/포함 관계에 있는 포인터는 초기화 하지 않음. 관계도가 바뀔 때 따로 변경
- Init와 중복되는 코드가 없어야 함.

Release/delete 규칙
- 꼭 필요한 상황이 아니라면 절대 원시 포인터 생성 금지
- 스마트 포인터는 Release에서 항상 reset
- 컨테이너, 배열에 스마트 포인터가 들어갈 경우 Release에서 항상 clear
- Manager, Scene을 제외한 모든 클래스는 소멸자에서 Release 호출

Enter/Exit 규칙
- 아직 존재하지 않는 함수
- 절대로 이곳에서 Init, Release하지 않음. Reset을 통해 준비할 수 있도록 코드 작성.
- 음악 전환이나 리소스를 Load, UnLoad하는 용도로 사용.

클래스 객체 컨테이너 규칙
- 스마트 포인터를 사용해 컨테이너에 담는다.
- 자신을 소유한 씬/매니저에서만 shared_ptr을 사용하고 다른 곳에선 weak_ptr을 사용한다.
	ex) ObjectTile은 SceneTest에서 만들어졌기 때문에 SceneTest만 ObjectTile의 shared_ptr을 사용할 수 있다.
